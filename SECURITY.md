# Security Policy

## 지원 버전

본 프로젝트는 활발히 개발 중이며, **최신 마이너 버전만 보안 패치를 받습니다**. v0.x 단계에서는 SemVer를 엄격히 보장하지 않으며, 보안 수정은 일반적으로 다음 패치 릴리즈로 묶어 배포합니다.

| 버전 | 지원 |
|------|:-----:|
| 0.2.x | ✅ 활발한 보안 패치 |
| < 0.2.0 | ❌ 미지원 (업그레이드 필요) |

v1.0 이후 LTS 정책이 도입될 예정입니다 — [ROADMAP.md](ROADMAP.md) 참고.

## 보안 정의

본 프로젝트의 컨텍스트에서 다음 항목들을 **보안 이슈**로 분류합니다:

### Critical (즉시 대응)
- 임의 코드 실행 (예: 악성 폰트 파일 처리 시 `subprocess` 호출 또는 deserialization 취약점)
- **글리프 외형 데이터 유출** — 추출기가 `glyf` / `CFF` / `CFF2` / `COLR` 등 outline 테이블을 읽거나 결과 폰트에 노출시키는 경로. 이는 본 프로젝트의 핵심 라이센스 안전 보장을 무너뜨리는 회귀입니다 ([docs/design/02-metrics-schema.md §라이센스 안전 경계](docs/design/02-metrics-schema.md#라이센스-안전-경계))
- 자격 증명 / 비밀 정보가 로그·결과 폰트의 `name` 테이블·메트릭 JSON 등으로 누출되는 경로

### High (다음 패치에서 우선 수정)
- 폰트 입력 파싱 시 발생하는 무한 루프 / DoS (메모리·CPU 고갈)
- 결과 폰트의 OpenType 구조 손상으로 다른 도구(브라우저, 디자인 툴)에서 충돌
- HFT/HWP 등 비공개 포맷 처리 추가 시 그 안의 잠재 임베디드 페이로드

### Medium / Low
- 정보 누출이지만 권한 상승으로 이어지지 않는 부수 효과 (예: 임시 파일이 너무 관대한 권한으로 생성)
- 의존성(`fontTools`, `uharfbuzz`) 자체의 알려진 취약점이 본 프로젝트 코드 경로에서 도달 불가능한 경우

## 신고 방법

**공개 GitHub 이슈에 보안 취약점을 게시하지 마세요.** 다음 채널 중 하나를 사용하세요:

### 권장: GitHub Private Vulnerability Reporting
1. 본 리포지토리 → Security → Advisories → "Report a vulnerability" 클릭
2. 영향 받는 버전, 재현 단계, 가능한 영향 범위(권한·범위) 기술
3. 제보자 정보(연락 가능 이메일) 포함

### 백업 채널
GitHub Private Vulnerability Reporting을 사용할 수 없는 환경이라면
[PolarisOffice 조직 페이지](https://github.com/PolarisOffice)의 공개 연락처를
이용하시고, 이메일 제목에 `[SECURITY] polaris_mcfg` 를 표기해주세요.

## 응답 SLA

| 단계 | 목표 시한 |
|------|----------|
| 접수 확인 | 영업일 기준 3일 내 |
| 초기 평가 (Critical/High/Medium/Low 분류) | 영업일 기준 7일 내 |
| 수정 패치 (Critical) | 14일 내 비공개 패치 후 공개 배포 |
| 수정 패치 (High) | 다음 마이너 릴리즈 |
| 수정 패치 (Medium/Low) | 마일스톤 일정에 맞춰 |
| 공개 advisory 발행 | 패치 배포 직후 또는 90일 후, 둘 중 빠른 시점 |

## 책임 있는 공개

제보자가 위 SLA 내에 응답을 받지 못한 경우, **90일 후 공개적 disclose** 권한을 가집니다 (Google Project Zero 식 표준 기간). 단, 협의를 통해 연장 가능합니다.

수정이 배포되면 advisory에 제보자 이름/핸들을 (동의 시) 명시합니다.

## 본 프로젝트가 다루지 않는 영역

- 사용자가 입력한 폰트의 EULA / 저작권 검토 — 본 도구는 메트릭 데이터만 다루지만, **소스 폰트의 라이센스 준수 책임은 사용자에게 있습니다**. EULA 위반은 보안 이슈가 아닌 라이센스 이슈로 분류합니다.
- 외부 서비스(GitHub, PyPI, 의존성 패키지) 자체의 취약점 — 해당 운영자에게 직접 신고
- 자체 신뢰 환경에서 사용자가 직접 변조한 폰트 파일을 실행하는 경우의 안전 — 이는 보안 모델 외(out-of-trust-model)
